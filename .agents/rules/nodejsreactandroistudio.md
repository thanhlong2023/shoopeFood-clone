Spring Boot Rules:

- Use constructor injection.
- Use DTO pattern.
- Use Service layer.
- Use Repository layer.
- Use Global Exception Handler.
- Use Bean Validation.
- Follow REST conventions.
- Use Spring Security best practices.
- Use JPA best practices.
- Generate JUnit 5 and MockMvc tests when needed.

Node.js Rules:

- Use layered architecture.
- Use Service Pattern.
- Use JWT Authentication.
- Use proper validation.
- Use centralized error handling.
- Follow Express/NestJS best practices.
- Generate Jest tests when appropriate.

React Rules:

- Use functional components.
- Use hooks.
- Avoid prop drilling.
- Use reusable components.
- Handle loading and error states.
- Use feature-based folder structure.
- Prefer TypeScript when possible.

Android Rules:

- Use MVVM architecture.
- Use Repository Pattern.
- Use ViewModel.
- Use StateFlow or LiveData.
- Use Retrofit for API calls.
- Use Room Database for local storage.
- Follow Clean Architecture.
- Separate UI, Domain, and Data layers.
- Use Dependency Injection when appropriate.
- Prefer Kotlin for new projects.
- Explain Android lifecycle when relevant.
